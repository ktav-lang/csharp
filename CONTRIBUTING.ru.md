# Участие в разработке Ktav (C# / .NET)

**Языки:** [English](CONTRIBUTING.md) · **Русский** · [简体中文](CONTRIBUTING.zh.md)

## Основные правила

### 1. Каждый баг-фикс сопровождается регрессионным тестом

Перед исправлением бага — напишите падающий тест. Тест и фикс в одном PR.

### 2. Не изобретайте формат в биндингах

Парсер и формат живут в Rust-крейте
([`ktav-lang/rust`](https://github.com/ktav-lang/rust)). Здесь — только
C#-специфичная эргономика.

### 3. Изменения публичного API — с пометкой совместимости

Укажите в PR, semver-совместимое изменение или semver-breaking.
Обновите `CHANGELOG.md` и два перевода в том же PR.

### 4. Один концепт — один коммит

Без `feat:` / `fix:` префиксов.

## Дев-сетап

Нужны .NET 8 SDK и Rust toolchain (MSRV 1.70).

```
cargo build --release -p ktav-ffi
dotnet test -c Release
```

Spec-conformance тесты резолвят spec через env / submodule / sibling fallback.
Если spec не найден — тесты скипаются.

## Языковая политика

Репо участвует в трёхъязычной политике организации (EN / RU / ZH).
Обновление — атомарное, во всех трёх версиях одновременно.
